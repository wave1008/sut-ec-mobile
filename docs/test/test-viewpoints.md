## テキスト版アウトライン

### 1. 機能観点
- **ホーム**: おすすめ/ベストセラー横スクロール表示、カテゴリチップ導線、検索バー導線、キャンペーンバナー表示。
- **検索・絞り込み**: キーワード検索、カテゴリ選択、価格帯4区分(〜1,000 / 1,000〜5,000 / 5,000〜20,000 / 20,000〜)、並び替え5種(おすすめ/価格昇順/価格降順/評価/新着)、0件結果と未検索状態の区別。
- **商品詳細**: 画像カルーセル+ドットインジケータ、価格と割引取消線、在庫あり/切れ表示、タグ表示、レビュー一覧/0件、関連商品、数量ステッパー、カート追加時のスナックバー("見る"導線)。
- **カート**: 数量増減、数量1→0での自動削除、ゴミ箱削除、小計/送料/合計、"あと◯◯円で送料無料"案内、空カート表示。
- **チェックアウト**: 住所選択、支払い選択、両方選択で「注文を確定する」活性化、確定でカート空、注文完了画面(注文番号/合計/お届け目安)。
- **注文履歴**: 一覧(新しい順)、注文詳細、ステータス4種(準備中/発送済み/配達済み/キャンセル)。
- **お気に入り**: 登録/解除、一覧、空状態。
- **認証**: ログイン(任意情報で成功)、サインアップ、ログアウト、未ログイン/ログイン時の表示差。
- **住所管理**: 一覧/追加/編集/削除/デフォルト設定。
- **支払い管理**: カード/代金引換、追加/編集/削除/デフォルト設定。

### 2. 状態同期観点
- カートバッジが全画面(下タブ)で即時更新される。
- お気に入り状態がホーム/検索/詳細/一覧で共通・即時反映。
- 言語切替が全画面に即時反映(再表示不要)。
- 注文確定後、注文履歴に即時追加されカートが空になる。

### 3. 国際化(i18n)観点
- 日英トグルで全 UI 文言が切り替わる。
- 商品名・説明・レビューが選択言語で表示される。
- 翻訳漏れ・固定表記(例: "Prime" は両言語同一)の妥当性。
- 送料案内など言語別の文言構造の差異。

### 4. 業務ルール観点
- 送料: 小計 3,000 円**以上**で無料、未満で 500 円、空カートで 0 円(境界値)。
- 税込表示のため tax=0、合計 = 小計 + 送料(税込のため独立した税行は表示しない)。
- 割引率 = (定価 − 価格) / 定価。
- 注文はカートのスナップショットで、確定後のカート変更に影響されない。

### 5. UI/UX 観点
- 画面遷移・戻る操作・下タブの表示/非表示切替。
- ローディング表示、空状態・未検索・未登録の各表示。
- ボタンの活性/非活性(在庫切れ、住所/支払い未選択、注文処理中)。

### 6. クロスプラットフォーム観点
- Android / iOS の双方で起動・描画・同一シナリオ成立。
- iOS 固有: `Info.plist` の `CADisableMinimumFrameDurationOnPhone=true` により起動時クラッシュしないこと。

### 7. 異常系・境界・例外観点(強化)
イレギュラー系は本節を重点とする。詳細は [テストシナリオ一覧 §14](test-scenarios.md#14-異常系境界例外強化) / [手順書 §14](test-procedures.md#14-異常系境界例外強化) を参照。

**(A) 入力バリデーション**
- ログイン/サインアップの**空・空白のみ**の資格情報 → 実装は `isBlank()` で失敗する。ただし正本 design(任意情報で成功)/ UI 文言「任意の情報でログインできます」/ 実装(空欄=失敗)が**三者不一致**。テストは実装挙動の再現を記録するに留め、**どちらが正か(仕様/UI 文言/実装のどれを直すか)はテストで独断確定せず起票事項として扱う**。
- 住所/支払いフォームの必須項目未入力・超長文・特殊文字・絵文字・非数字(郵便番号/電話/カード番号)・過去日付の有効期限。
- 検索の空白のみ(trim される)・超長文・特殊文字。

**(B) 境界値**
- 送料無料境界: 小計 2,999 / **3,000(ちょうどで無料)** / 3,001 円。
- 価格帯フィルタ境界: ちょうど 1,000 / 5,000 / 20,000 円の商品がどちらの帯に入るか。
- 数量: 詳細画面の下限(1 未満に減らせない)、カートの大量数量と行小計/合計の桁。

**(C) 空・未登録状態**
- 住所を全削除 → チェックアウトで「住所未登録」表示・確定不可。
- 支払いを全削除 → 「支払い未登録」表示・確定不可。
- デフォルト住所/支払いを削除 → デフォルト不在時の初期選択の挙動。
- レビュー0件・検索0件・お気に入り空。

**(D) 状態・並行・遷移(重点)**
- 「注文を確定する」連打 → 二重注文されない(`placing` ガード)。
- **チェックアウトで選択中の住所/支払いを「管理」から削除して戻る** → 選択整合性。確定ボタンの活性状態と、押下時に無反応にならないか(**潜在リスク領域**)。
- **同一 CheckoutViewModel 保持中に住所/支払い一覧が変化する任意経路**(削除だけでなく、デフォルト住所の削除・全削除・別タブ経由の編集後にチェックアウトへ戻る等)でも、選択 ID が実在しないまま保持され確定ボタンがサイレント失敗しないか。★の根本原因は選択 ID の再検証欠如のため「管理から削除」経路に限らず同型に再現する。
- 注文処理中(placing)の戻る/言語切替。
- 商品詳細→関連商品→…の深い遷移からの戻る。
- 未ログインのままの注文可否。ログアウト時のカート/注文履歴の扱い。
- ローディング中の多重タップ・高速タブ切替。

**(E) ネットワーク/ライフサイクル**
- 画像取得失敗(**ローカル画像サーバー `:8000` を停止**して注入。画像はローカル配信のため機内モードでは再現不可)時に画像以外の操作が継続できる(クラッシュしない)。
- 通信擬似 `delay(200–600ms)` 中の操作(ロード中の戻る等)。
- バックグラウンド→復帰、画面回転での状態保持・レイアウト崩れ。
- アプリ再起動でデータが初期化される前提の確認。
