# sut-ec-mobile テスト手順書

各テストケース(TC-xx)は [テストシナリオ一覧](test-scenarios.md) の SC-xx に対応する。特記なき限り**両プラットフォーム(Android / iOS)**で実施する。

## 共通事項

> **第2段階での差分**: 本手順書は第1段階（モック）前提。実サーバー接続版の差分は [テスト計画書 §0.1](test-plan.md#01-第2段階サーバー実装でのテストへの影響) を各 TC に上書き適用すること（前提環境・認証・初期データ・注文番号などが変わる）。

### 前提条件(全 TC 共通)
- アプリがビルド・インストール済みで、エミュレータ/シミュレータで起動できる。
- **サーバーが起動**している（第2段階）: `./scripts/dev-server.sh`（Apple Container で PostgreSQL 起動 → `:server` 実行）。既定 Android=`10.0.2.2:8090` / iOS=`127.0.0.1:8090`（設定可能）。停止時は商品・画像とも表示されない。〔第1段階は python 画像サーバー :8000 だった〕
- 認証系・ユーザーデータ系 TC は**事前にアカウント登録/ログイン**しておく（第2段階は実認証・ユーザー単位保存）。新規アカウントは住所/支払い/注文が空なので、チェックアウト系 TC は先に住所・支払いを登録する。
- データは永続（第2段階）。第1段階の「1 起動セッション内で完結」制約は不要。〔第1段階は非永続〕

### 初期シードデータ(判定の基準値)
- **商品例**: 「ワイヤレスイヤホン Pro / Wireless Earbuds Pro」= ¥8,900(定価 ¥12,000、割引あり)、カテゴリ=家電、タグ=ベストセラー/Prime、レビューあり。
- **低価格商品**: 「USB-C 充電ケーブル 2m」= ¥500(送料境界の検証に使用)。
- **カテゴリ**: 8 種(家電・電化製品 / ファッション / ホーム・キッチン / ビューティー / 本 / スポーツ・アウトドア / 食品・飲料 / おもちゃ)。
- **送料ルール**: 小計 **3,000 円以上で無料**、未満は **500 円**、空カートは 0 円。税込(tax=0)。
- **初期住所**: 「山田 太郎(東京都渋谷区)」=デフォルト、「山田 太郎(大阪府)」。
- **初期支払い**: 「Visa •••• 1234」=デフォルト、「代金引換」。
- **初期注文**: `ord_1`=配達済み(2026-06-02) / `ord_2`=発送済み(2026-06-25) / `ord_3`=準備中(2026-07-10)。

**前提 SKU の確定(実施/判定の再現性のため固定する)**:
- **複数画像商品(TC-31)**: 「ワイヤレスイヤホン Pro(electronics_1)」= 画像4枚。カルーセル+ドット追従の確認に使用。
- **レビュー0件商品(TC-35)**: 「ウールマフラー(fashion_4)」= レビュー明細0件で「レビューはまだありません」表示。※注意: 商品には `reviewCount`(件数バッジ)が付くが、レビュー明細一覧は本商品では空。バッジ件数>0 と明細0件の不整合が見えたら実挙動を記録し起票検討。
- **送料境界の到達組合せ(TC-140)**: ¥2,980 = beauty_1 ×1 / ¥3,000 = electronics_7 ×6 または grocery_1+grocery_2。詳細は TC-140 参照。
- **在庫切れ商品(TC-37)**: **シードに `inStock=false` の商品は存在しない**(全商品が在庫あり)。よって「在庫切れで『カートに追加』非活性」は**現行シードでは実施不能**。実施するにはシードへ `inStock=false` 商品の追加が必要(テスト用データ整備事項として起票)。`LOW_STOCK` タグ(sports_3)は表示タグであり在庫切れ=非活性とは別物。

### 結果記入欄の凡例
各 TC の結果は `☐ Pass / ☐ Fail` に記録し、Fail 時は再現手順・スクリーンショット・重大度を不具合一覧へ起票する。

### TC-32(SC-32)割引表示 【中】
割引のある商品で、割引後価格が強調表示され、定価が取消線で併記される。加えて**割引率(%)の算出表示**(`-N%` 形式)を確認する。算出は [PriceText.kt:37](../../composeApp/src/commonMain/kotlin/com/sutec/mobile/designsystem/component/PriceText.kt#L37) の `((定価 − 価格) * 100) / 定価` = **整数除算による切り捨て(小数点以下切り捨て、四捨五入ではない)**。例: electronics_1 = (12,000−8,900)×100/12,000 = 25.83… → **-25%**、electronics_3 = (15,000−12,500)×100/15,000 = 16.67… → **-16%**(四捨五入した 26% / 17% は誤り。表示は切り捨て値)。表示値が切り捨て算出値と一致すること。 ☐ Pass / ☐ Fail

### 14.2 境界値

#### TC-140(SC-140)送料境界(3,000 円前後)【高】
**前提**: シード商品の価格は**すべて 10 円単位(奇数円が存在しない)**。したがって小計を**ちょうど 2,999 / 3,001 円に到達させることは不可能**。境界の直下代表として到達可能な最大値 **¥2,980** を用い、境界ちょうど **¥3,000** と対比する(判定基準は「**3,000 以上=無料 / 未満=¥500**」)。

| # | 操作(小計を調整) | 期待結果 |
|---|---|---|
| 1 | 「ビタミンC美容液(beauty_1)」¥2,980 を1点だけカートに入れる(小計 **¥2,980**、3,000 円未満の直下) | 送料 = ¥500、「あと¥20で送料無料」の案内あり、合計 = ¥3,480 |
| 2 | 小計を **ちょうど ¥3,000** にする(下記いずれか) | 送料 = **無料**、案内が消える。合計 = ¥3,000 |
| 3 | 小計を **¥3,000 超**にする(例: 「ウールマフラー(fashion_4)」¥3,200 を1点) | 送料 = 無料 |

**¥3,000 ちょうどの作り方(いずれか)**:
- 「USB-C 充電ケーブル 2m(electronics_7)」¥500 × **6 点** = ¥3,000(単一 SKU で最も簡単)。
- 「高級煎茶 100g(grocery_1)」¥1,200 × 1 + 「素焼きミックスナッツ 1kg(grocery_2)」¥1,800 × 1 = ¥3,000。

**補足**: 2,999/3,001 が作れない旨は仕様上の制約(整数円・10 円単位シード)であり不具合ではない。境界の本質(**3,000 以上=無料**)は上記 ¥2,980 / ¥3,000 の対比で判定できる。 ☐ Pass / ☐ Fail
